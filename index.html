Branch access & role

Controls which branches a user can see. Owner/Admin sees all; Branch manager sees assigned branches; Employee sees only their own. The active branch persists across refresh.
This is a front-end (UI) access control for demonstration. It is not a security boundary — real enforcement requires the server-side branch scoping described in the deployment notes. Anyone with the file can change roles.

What changed in V1

Removed for simplicity: Average Basket, Average Margin, Customer Management, Loyalty. The system is now accounting-first: sales, expenses, purchases, tax, payroll, reports and inventory — driven by the Business Assistant and Telegram.
